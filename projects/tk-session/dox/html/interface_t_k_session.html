- (void) componentDidBegin: (NSNotification *)  info

The component instance did begin.

This method is to be considered unreliable. If you need to know when a component is to begin, use componentWillBegin, which will always be sent before the component begins, while this method, may return much later than expected due to issues with the way the components run loop is handled.

- (void) componentWillBegin: (NSNotification *)  info

The current component is about to begin.

This should be used rather than componentDidBegin due to reliability issues.
